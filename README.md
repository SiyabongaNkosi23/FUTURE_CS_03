## Overview
This project focuses on analyzing a public REST API to identify common security risks, authentication weaknesses, and data exposure vulnerabilities.

## Objectives
- Identify insecure API endpoints
- Analyze authentication and authorization issues
- Detect data exposure risks
- Recommend security improvements

## Tools Used
- Browser
- Postman
- Screenshot Tools
- GitHub

## Key Findings
- No authentication required
- Public access to API data
- Lack of access control
- Potential data exposure risks

## Recommendations
- Implement JWT or API key authentication
- Restrict sensitive data exposure
- Add role-based access control
- Apply rate limiting to prevent abuse
